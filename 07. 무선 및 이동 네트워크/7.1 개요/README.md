# 7.1 개요

## 무선 네트워크 소개

무선 데이터 통신과 이동성을 다루며, WiFi, 4G, 5G 셀룰러 네트워크를 포함한 다양한 네트워크를 살펴본다.

### 1. 무선 호스트 (Wireless Hosts)

유선 네트워크와 마찬가지로 애플리케이션을 실행하는 종단 시스템 장치이다.

| 장치 유형 | 예시 |
|----------|------|
| **모바일 장치** | 스마트폰, 태블릿, 노트북 |
| **IoT 장치** | 센서, 가전제품, 자동차 |
| **기타** | 인터넷에 연결되는 수많은 장치들 |

> 호스트는 이동할 수도 있고, 이동하지 않을 수도 있다.

### 2. 무선 링크 (Wireless Links)

호스트가 기지국(또는 다른 무선 호스트)에 **무선 통신 링크**를 통해 연결된다.

| 특성 | 설명 |
|------|------|
| **전송률** | 무선 링크 기술마다 다름 |
| **전송 거리** | 기술에 따라 다양한 거리 지원 |
| **비트 오류율** | 유선보다 높음 (7.2절에서 다룸) |

### 3. 기지국 (Base Station)

무선 네트워크 인프라의 핵심 부분이다.

| 특성 | 설명 |
|------|------|
| **역할** | 무선 호스트와 데이터(패킷) 송수신 담당 |
| **조정 기능** | 연결된 다수의 무선 호스트의 전송 조정 |
| **중계 역할** | 호스트와 더 큰 네트워크 간의 링크 계층 중계 |

**기지국의 예**:
- **셀 타워 (Cell Tower)**: 셀룰러 네트워크
- **액세스 포인트 (Access Point)**: 802.11 무선 LAN

**"연결(Associated)"의 의미**:
1. 호스트가 기지국의 무선 통신 거리 내에 있음
2. 호스트가 해당 기지국을 사용하여 더 큰 네트워크와 데이터 중계

### 4. 네트워크 인프라 (Network Infrastructure)

무선 호스트가 통신하고자 하는 더 큰 네트워크 (예: 인터넷, 기업망, 가정망)

## 무선 전송률 및 범위

| 표준 | 전송률 | 범위 | 거리 |
|------|--------|------|------|
| **802.11ax** | 14 Gbps | 실내 | 10-30m |
| **5G** | 10 Gbps | 중거리 실외 | 200m-4Km |
| **802.11ac** | 3.5 Gbps | 실내 | 10-30m |
| **802.11 af,ah** | - | 실외 | 50-200m |
| **802.11n** | 600 Mbps | 실내 | 10-30m |
| **4G LTE** | - | 장거리 실외 | 4Km-15Km |
| **802.11g** | 54 Mbps | 실내 | 10-30m |
| **802.11b** | 11 Mbps | 실내 | 10-30m |
| **Bluetooth** | 2 Mbps | 실내 | 10-30m |

> 참고: 일부 네트워크는 현재 배포 중이며, 거리, 채널 조건, 사용자 수에 따라 링크 속도가 변할 수 있다.

## 인프라 모드 vs 애드혹 네트워크

### 인프라 모드 (Infrastructure Mode)

- 기지국과 연결된 호스트가 운영하는 방식
- 모든 전통적인 네트워크 서비스(주소 할당, 라우팅)가 기지국을 통해 연결된 네트워크에서 제공

### 애드혹 네트워크 (Ad Hoc Networks)

- 연결할 인프라가 없는 무선 호스트
- 호스트 자체가 다음 서비스를 제공해야 함:
  - 라우팅
  - 주소 할당
  - DNS 유사 이름 변환

---

## 핸드오프 / 핸드오버 (Handoff / Handover)

모바일 호스트가 한 기지국의 범위를 벗어나 다른 기지국의 범위로 이동할 때 발생한다.

### 핸드오프의 과제

| 과제 | 설명 |
|------|------|
| **위치 파악** | 이동 호스트의 현재 위치를 어떻게 찾는가? |
| **주소 지정** | 호스트가 여러 가능한 위치 중 하나에 있을 때 주소 지정은? |
| **연결 유지** | TCP 연결이나 전화 통화 중 호스트가 이동하면 데이터 라우팅은? |

> 이러한 질문들이 무선 및 이동 네트워킹을 흥미로운 연구 영역으로 만든다.

## 무선 네트워크 분류

무선 네트워크를 두 가지 기준으로 분류할 수 있다:
1. 패킷이 **단일 무선 홉**을 거치는지 **다중 무선 홉**을 거치는지
2. 네트워크에 기지국과 같은 **인프라**가 있는지 여부

### 분류 체계

| 유형 | 인프라 | 홉 수 | 설명 | 예시 |
|------|--------|-------|------|------|
| **단일 홉, 인프라 기반** | O | 1 | 기지국이 더 큰 유선 네트워크에 연결됨. 모든 통신이 기지국과 무선 호스트 간 단일 무선 홉으로 이루어짐 | 교실/카페/도서관의 802.11 네트워크, 4G LTE |
| **단일 홉, 인프라 없음** | X | 1 | 기지국 없음. 단일 홉 네트워크의 한 노드가 다른 노드들의 전송을 조정 | Bluetooth (키보드, 스피커, 헤드셋 연결) |
| **다중 홉, 인프라 기반** | O | 다수 | 기지국이 유선 네트워크에 연결됨. 일부 무선 노드가 기지국과 통신하기 위해 다른 무선 노드를 통해 중계해야 함 | 무선 센서 네트워크, 무선 메시 네트워크 |
| **다중 홉, 인프라 없음** | X | 다수 | 기지국 없음. 노드가 목적지에 도달하기 위해 여러 다른 노드를 통해 메시지 중계. 노드가 이동 가능하여 연결성이 변함 | 모바일 애드혹 네트워크 (MANET), 차량 애드혹 네트워크 (VANET) |
