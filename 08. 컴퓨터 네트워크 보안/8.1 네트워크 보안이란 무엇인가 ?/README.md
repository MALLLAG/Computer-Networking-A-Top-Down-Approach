## 8.1 네트워크 보안이란 무엇인가?

네트워크 보안에 대한 연구를 시작하기 위해, 통신하고자 하는 두 사람인 **앨리스(Alice)** 와 **밥(Bob)** 을 소개하겠습니다.
"보안(securely)"이란 정확히 무엇을 의미할까요?

앨리스가 밥에게만 보내는 메시지를 밥만 이해할 수 있기를 원한다고 해도, 두 사람이 안전하지 않은 매체를 통해 통신하고 있다면 침입자가 개입할 수 있습니다.
**침입자(intruder)** 인 **트루디(Trudy)** 는 앨리스가 밥에게 전송하는 메시지를 가로채거나, 수정하거나, 삭제할 수 있습니다.

이러한 상황을 고려할 때, 다음과 같은 **안전한 통신(secure communication)** 의 바람직한 속성들을 식별할 수 있습니다.

### 안전한 통신의 네 가지 속성

#### 기밀성 (Confidentiality)

송신자와 의도된 수신자만이 전송된 메시지의 내용을 이해할 수 있어야 합니다.
도청자가 메시지를 가로챌 수 있기 때문에, 메시지가 어떤 방식으로든 **암호화(encrypted)** 되어 가로챈 메시지를 도청자가 이해할 수 없도록 해야 합니다.

이러한 기밀성 측면이 **안전한 통신(secure communication)** 이라는 용어에서 가장 일반적으로 인식되는 의미입니다.
8.2절에서 데이터 암호화 및 복호화를 위한 암호화 기술을 학습합니다.

#### 메시지 무결성 (Message Integrity)

앨리스와 밥은 통신 내용이 전송 중에 악의적으로든 우연히든 **변경되지 않았음을 보장**받기를 원합니다.
신뢰할 수 있는 전송에서 살펴본 체크섬 기술의 확장을 통해 메시지 무결성을 제공할 수 있습니다.

8.3절에서 메시지 무결성에 대해 학습합니다.

#### 종단점 인증 (End-point Authentication)

송신자와 수신자 모두 통신에 관련된 **상대방의 신원을 확인**할 수 있어야 합니다.
즉, 상대방이 실제로 그들이 주장하는 사람임을 확인할 수 있어야 합니다.

서로를 볼 수 없는 매체를 통해 통신할 때 인증은 간단하지 않습니다.
예를 들어, 사용자가 받은 편지함에 접근하려 할 때 메일 서버는 그 사용자가 실제로 주장하는 사람인지 어떻게 확인할까요?

8.4절에서 종단점 인증에 대해 학습합니다.

#### 운영 보안 (Operational Security)

오늘날 거의 모든 조직(기업, 대학 등)은 공용 인터넷에 연결된 네트워크를 가지고 있습니다.
따라서 이러한 네트워크는 잠재적으로 **침해될 수 있습니다**.

공격자는 네트워크 호스트에 웜(worm)을 심거나, 기업 비밀을 획득하거나, 내부 네트워크 구성을 파악하거나, **DoS 공격(Denial of Service attack)** 을 시도할 수 있습니다.

8.9절에서 **방화벽(firewall)** 과 **침입 탐지 시스템(intrusion detection system)** 과 같은 운영 장치들이 조직의 네트워크에 대한 공격을 방어하는 데 사용됨을 살펴봅니다.

### 송신자, 수신자, 침입자

**그림 8.1** 은 앨리스, 밥, 트루디의 관계를 보여줍니다.

- **앨리스(Alice)**: 데이터를 안전하게 보내고자 하는 송신자
- **밥(Bob)**: 데이터를 수신하는 수신자
- **트루디(Trudy)**: 통신을 가로채거나 방해하려는 침입자

앨리스가 밥에게 데이터를 보낼 때, 기밀성, 종단점 인증, 메시지 무결성의 요구사항을 충족하면서
앨리스와 밥은 TCP가 송신자와 수신자 사이에 제어 세그먼트와 데이터 세그먼트를 교환하는 것과 유사한 방식으로
**제어 메시지와 데이터 메시지를 교환**합니다.

### 침입자의 공격 방법

1.6절에서 논의한 것처럼, 이러한 메시지 전부 또는 일부는 일반적으로 **암호화**됩니다.
침입자는 잠재적으로 다음과 같은 공격을 수행할 수 있습니다:

- **도청(eavesdropping)**: 채널에서 제어 메시지와 데이터 메시지를 스니핑하고 기록
- **수정, 삽입 또는 삭제(modification, insertion, or deletion)**: 메시지 또는 메시지 내용의 변조

적절한 대응책이 취해지지 않으면, 이러한 능력으로 인해 침입자는 다양한 보안 공격을 수행할 수 있습니다:

- 통신 스누핑(비밀번호와 데이터 탈취 가능)
- 다른 개체 사칭
- 진행 중인 세션 하이재킹
- 시스템 자원 과부하로 정당한 네트워크 사용자에게 서비스 거부

보고된 공격의 요약은 CERT 조정 센터(CERT Coordination Center)에서 관리되고 있습니다 [CERT 2020].

### 인터넷에서 안전한 통신이 필요한 실제 사례

인터넷에서 실제로 위협이 존재한다면, 안전하게 통신해야 하는 앨리스와 밥에 해당하는 인터넷 상의 실체는 누구일까요?

- **실제 사용자**: 안전한 이메일을 교환하고 싶은 실제 앨리스와 밥
- **전자 상거래 참여자**: 신용카드 번호를 웹 서버로 안전하게 전송하고 싶은 사용자
- **온라인 뱅킹 사용자**: 은행과 안전하게 상호작용하고 싶은 사용자
- **네트워크 인프라 자체**: 라우팅 정보를 교환하는 DNS나 라우팅 데몬도 안전한 통신이 필요

DNS 조회(2.4절 참조), 라우팅 계산(5.3절, 5.4절 참조), 네트워크 관리 기능(5.5절, 5.7절 참조)을 적극적으로 방해할 수 있는 침입자는 인터넷에 큰 혼란을 일으킬 수 있습니다.

### 암호학의 중요성

프레임워크와 네트워크 보안의 가장 중요한 정의들, 그리고 필요성을 확립했으니, 이제 **암호학(cryptography)** 에 대해 알아봅니다.

기밀성 제공에 암호학의 사용이 자명한 것처럼, 암호학은 **종단점 인증**과 **메시지 무결성** 제공에도 핵심적입니다.
이로 인해 암호학은 **네트워크 보안의 초석(cornerstone)** 이 됩니다.
